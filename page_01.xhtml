Developer Talk

When creating courses in Super Mario Maker, we start off in exactly the same way we used to when making courses for the original Super Mario Bros.: by picturing the course in our heads. Thinking up a new course has always been a fun process. That feeling never changes.

However, it's the next step where things have changed a lot. In Super Mario Maker, we can just tap to place the parts wherever we like when creating courses, but back in the days of the original Super Mario Bros., the course designer had to draw up a whole diagram of where all the enemies and blocks should be placed on graph paper.

We'd give that diagram to the programmer, and the programmer would convert all the enemy and block positions into numbers that he could enter into his computer.
Once all the numbers were entered, the computer would start creating the data to display them on the game screen. And once the data was complete, we had to write it to the semiconductor memory called a "ROM."
We would insert that ROM into our special developers' game cartridges, and insert them into an actual NES™ console—only then could we see the course displayed on the TV screen.

As you can see, it took quite a long time before we could even see a course on-screen, so it wasn't possible to keep constantly fixing mistakes or reworking things bit by bit.
As technology progressed, we reached the stage where the programmer could put the whole course together right there on his computer, and the process of creating a ROM became much quicker, so we were able to create lots of different versions of courses and try things out until we had the perfect layout.
This was the point where the "fun of thinking up a course" turned into the "fun of actually creating and trying it out."

But of course, there were still limitations. We couldn't do any of this on an ordinary game console—we had to use special development consoles combined with special software on high-spec computers.
And this is still true today—the whole Mario series has been created using this process.

So we started thinking, "Is there any way we could share this fun process with the players? If we use the Wii U GamePad, it feels like it could work..."

...And it did! (Yep, it's that simple!)
We managed to create a product that lets anyone create Mario courses on their own Wii U.

"Now, everyone can experience the fun of creating! ...But no, we can't leave it at that. Right now, it's still just a course-creation tool. It needs something else before we can release it as a game!"

What was missing was the "element of surprise." So we worked our hardest to turn that basic tool into an experience full of weird and wonderful things that had never been seen in the Mario series before—and that's how Super Mario Maker was born!
